
## 0. 文档状态

**已确认的设计决定：**

- 刺激设计以目标节点周围的"根壳结构"（一环—二环—三环）为主轴组织，而非按总节点数划分。
- 旧分析中继承的 `preferred_color_template` 不可直接沿用——其中存在相邻节点最大概率颜色相同的情况，不满足正常着色约束。
- 证据强度（evidence strength）固定为三档：`high / medium / low`。

**当前已给出默认接口、但后续仍可细化的事项：**

- 人类实验界面的具体视觉与交互实现。
- 被试（participant）级别 belief report 的具体观测模型参数化形式。
- 合成模型恢复（synthetic model recovery）的最终图表组织与展示细节。

## 1. 刺激集筛选

### 1.1 目标

以根壳结构（rooted-shell）语言为框架，构造一组合法、可解释、具有诊断力的候选刺激，并从中选出最终进入 analysis-5 的 30 项最优刺激面板（top-30 panel）。

此处的"合法"包含两个层面：

- **结构层面**：每条刺激须由目标节点周围前三环的根壳模板组织。
- **颜色层面**：非目标节点的最大概率颜色模板必须满足正常着色约束——相邻节点不得共享同一最大概率颜色。

### 1.2 候选条件的基本构成

每个候选条件是一个三元组：

1. 一张基础图（即按 rooted-shell 原则新生成的结构图）；
2. 一个合法的标准颜色模板（canonical argmax-colour template）；
3. 一个证据强度分配方案（对每个非目标节点赋予 `high / medium / low`）。

后续的评分、排序、去重与面板筛选均以该三元组为基本单位，而非仅以抽象结构模板或证据模板为单位。换言之，analysis-5 真正操作的对象不是“一个结构类”，而是“某张基础图及其对应的颜色模板与强度配置”。

### 1.3 结构空间：按前三环组织，而非按总节点数穷举

刺激空间的组织以目标节点周围的前三环结构为框架：

- 一环：距目标节点为 1 的节点集合；
- 二环：距目标节点为 2 的节点集合；
- 三环：距目标节点为 3 的节点集合。

每个结构模板至少记录以下属性：

- 各环节点数（shell occupancy profile）
- 环间连接模式（shell transition template）
- 中继路径冗余度（relay redundancy）
- 环支撑类型（loop support class）
- 三环外是否有更远结构（`tail_beyond_third_shell`）

三环之外的更远结构可作为附加标记保留，但不应因此退回到"按总节点数枚举全部图"的方式。

**生成策略的优先级：**

- 优先覆盖结构差异，而非优先扩张节点数。
- 在能覆盖同等结构差异的前提下，优先保留节点数更少的图。
- 仅当较小节点数无法表达目标结构差异时，才增加节点数。
- 基础图的总节点数上限暂定为 10。

基础图集的目标是：在根壳结构上实现充分的覆盖，同时控制节点数量。当前建议将基础图集规模控制在约 300 张（此处的"基础图集"指尚未附加颜色模板与强度分配的纯结构图）。这一数量是工程可行性与结构覆盖度之间的折中，而非必须严格命中的精确指标。

这里所说的"基础图集"必须是 analysis-5 按当前 rooted-shell 设计重新生成的三环结构图集，而不是对既有图集沿用原图后再重新打 condition 标签。旧分析中的图库最多只能作为比对材料或生成参考，不能直接充当本轮基础图集。

**节点数偏好（软规则）：**

- 优先保留 5–7 节点的图。
- 8 节点图仅在其表达新的根壳结构差异时纳入。
- 9–10 节点图仅用于表达较小图无法呈现的关键结构差异。
- 若大小图在结构诊断上等价，保留较小者。

**结构覆盖原则：**

- 各类指标——环节点数分布、环间连接模式、中继冗余度、环支撑类型、有无三环外尾部——在基础图集中不得空缺。
- 每个主要结构类至少有两张代表图，避免单一样例承担整类。
- 常见且预期诊断性高的结构类分配更多代表图；稀有但理论上关键的结构类可保留较少代表图，但不得空缺。
- 若总量逼近 300，优先压缩"结构信息重复但节点数更大"的候选，而非压缩结构上独特的候选。

### 1.4 颜色模板：先合法化，再选标准代表

本分析不再沿用旧分析遗留的颜色模板。新的颜色模板以每张基础图为单位重新生成，流程如下：

1. 对给定基础图，枚举所有合法的颜色模板（"合法"指所有相邻非目标节点的最大概率颜色互不相同）；
2. 对仅相差颜色重命名的模板做等价压缩；
3. 对由图自同构导致的等价模板做进一步压缩；
4. 在压缩后的合法模板集合中，为该基础图选出一个标准颜色模板。

标准颜色模板并非对历史模板的简单继承，而是在合法模板空间中重新选出的代表；选择粒度为"每张基础图"，而非"每个抽象结构模板"。

### 1.5 标准颜色模板的选择原则

选择逻辑应借鉴 analysis-2 / analysis-3 中的自适应冲突（`adaptive conflicting`）思路，而非直接照搬旧模板。

建议流程：

1. 对每张基础图，在其合法颜色模板集合中逐一评估。
2. 评分分两阶段：
   - 第一阶段：分别用同质的 `high / medium / low` 强度档位评估各合法颜色模板；
   - 第二阶段：使用 analysis-2/3 中已有的指标综合排序，包括 `monotone_all_levels`、`all_pairwise_total`、`adjacent_total`、`separable_high`。
3. 对每张基础图保留一个最优颜色模板，作为其标准颜色模板。
4. 标准颜色模板冻结之后，方可展开异质强度分配。

`fixed conflicting` 模式可作为基线对照或合理性校验（sanity check），但不作为主选逻辑，也不进入最终的筛选链路。正式的颜色模板选择应采用"每张基础图各选一个最优合法模板"的自适应版本。

### 1.6 证据强度分配：建立在合法颜色模板之上

默认生成顺序严格为：

**基础图 → 标准颜色模板 → 证据强度分配**

即基础图与标准颜色模板都确定后，才对每个非目标节点分配证据强度。

当前强度档位固定为：

| 档位   | 参数值               | 含义       |
| ------ | -------------------- | ---------- |
| High   | `(0.9, 0.05, 0.05)`  | 高确定性   |
| Medium | `(0.7, 0.15, 0.15)`  | 中等确定性 |
| Low    | `(0.5, 0.25, 0.25)`  | 低确定性   |

强度分配的作用是调节颜色偏好的强弱程度，而非重新决定偏好哪种颜色。因此它必须建立在合法的颜色模板之上，不能反过来修补不合法的着色。

### 1.7 证据强度分配的生成方式

生成优先级如下：

1. 对给定基础图及其标准颜色模板，先对 `high / medium / low` 组合做对称性压缩；
2. 压缩后组合数仍可控时，默认枚举全部组合；
3. 仅当对称性压缩后组合数仍然过大时，才引入受控采样；
4. 若引入采样，须固定随机种子，并在输出文件中记录采样规则与候选总数。

不建议将纯随机采样作为默认生成方式。

### 1.8 去重规则

至少按以下四个层次逐级去重：

1. **结构等价**：基础图的根壳结构是否相同。
2. **颜色模板等价**：标准颜色模板是否相同。
3. **强度分配等价**：各非目标节点的强度档位是否相同。
4. **对称性合并**：颜色重命名与图自同构导致的等价条件。

换言之，两个条件只有在结构、标准颜色模板、强度分配三者均等价时，才视为同一候选条件。

### 1.9 评分指标

完成颜色模板与强度分配后，对每个候选条件计算以下三项主指标：

这里的“不同族”必须明确指 analysis-5 继承自 Analysis 4 的 6-family 候选 agent 集合：

- `M_exact`
- `M_0`
- `M_rho`
- `M_rho,beta`
- `M_star`
- `M_cluster`

也就是说，下面所有评分都以这 6 个候选 agent 在目标节点上的预测差异为对象，而不是以 analysis-2 / analysis-3 中的 `A0...A4` rho-spectrum 为主对象。`A0...A4` 仍可作为补充分析输出，但不进入本部分的主评分定义。

#### 主评分

主评分以 6-family 的总体可分性为主，而不是以单一极端 pair 为主。默认主评分包括：

1. **族分离度**（family separation）：衡量上述 6 个候选 agent 在目标节点后验分布上的差异程度。默认使用有界的对称 Jensen-Shannon 散度（JS）作为主度量，并对 6-family 的所有 pair 做汇总；
2. **最大概率颜色分歧**（argmax disagreement）：检查上述 6 个候选 agent 在目标节点上预测的最可能颜色是否一致；
3. **后验非退化性**（posterior non-degeneracy）：排除后验过平（近似均匀，无法形成有效判断）或过尖（近似确定，族间差异难以体现）的刺激。

analysis-5 当前明确采用“6-family 两两比较为主”的方案，而不是继续沿用 analysis-2 / analysis-3 中那种以单一极端 agent pair 为核心的筛选逻辑。

#### 辅助指标

除主评分外，还应导出若干 pair-specific 辅助指标，用于检查当前最可能混淆的 family 边界是否被有效拉开。当前至少跟踪以下三组困难边界：

- `M_exact` vs `M_cluster`
- `M_rho` vs `M_rho,beta`
- `M_0` vs `M_star`

这些辅助指标不取代主评分，但应用于：

- 识别哪些刺激虽然总体 separation 较高，却仍然在关键 family 边界上无力；
- 帮助解释 confusion matrix 中最主要的混淆来源；
- 指导最终 panel 在困难 family 边界上的补偿性覆盖。

如需补充，可继续导出对称 KL 散度作为辅助输出，但不取代上述主评分地位。当前明确采用“JS 为主、KL 为辅”的方案。

### 1.10 最优面板的组装原则

最终面板固定为 30 项条件（从候选空间中筛选出的最终条件面板，而非基础图集大小）。

组装采用两阶段流程，而非直接取全局得分最高的前 30 条：

**阶段 A——评分预选**：依据族分离度、最大概率颜色分歧、后验非退化性对所有条件做全局排序，得到候选池。

**阶段 B——多样性组装**：从候选池中组装最终面板，在保持高平均分的同时，覆盖以下结构多样性：

- 不同环节点数分布
- 不同环间连接模式
- 不同中继冗余度与环支撑类型
- 包含/不包含三环外尾部结构

多样性约束直接作用于最终条件面板，不要求与基础图集配额完全一一对应；但若某类结构在基础图集里本就极度稀缺，须在面板报告中予以说明。

### 1.11 本部分应导出的文件

- 基础图库概览（base graph library summary）
- 根壳结构模板目录（rooted-shell structure template catalog）
- 合法颜色模板目录（legal colour-template catalog）
- 标准颜色模板选择记录（canonical colour-template selection summary）
- 证据强度候选方案汇总（strength-assignment candidate summary）
- 最终条件评分排序表（final condition ranking table）
- 前 30 项条件面板文件（top-30 panel file）

## 2. 人类实验任务设计

### 2.1 本部分的定位

本轮 analysis-5 的主任务仍是刺激集生成、条件筛选与 synthetic recovery，因此这里不展开完整的人类实验实施方案，而只保留一层最小接口说明。其目的不是提前锁死实验细节，而是确保当前筛选出的刺激、后续的 recovery，以及未来真实被试数据的拟合方式彼此兼容。

### 2.2 暂定的正式任务形式

当前建议将后续人类实验的正式任务暂定为一个连续 belief 报告任务，而不是离散三选一判断。

在每个 trial 中，被试看到：

- 一个围绕目标节点组织的 rooted-shell 图；
- 非目标节点上的颜色线索与强度线索；
- 一个待报告 belief 的目标节点。

被试的任务是：根据图中其他节点提供的线索，报告自己对目标节点颜色分布的主观判断。默认界面设想为一个三维颜色 belief ring 或等价的 simplex 控件，被试通过拖动一个颜色指针，给出目标节点在三种颜色之间的相对权重，而不是只提交一个确定的单一颜色。

这里的连续 belief 报告是本轮 analysis-5 默认服务的任务接口。只要这一接口不改变，前面的刺激筛选逻辑与后面的 recovery、拟合都可以保持一致。若未来再退回离散颜色判断，或改为其他形式的 richer response format，则应视为后续变体，而不是本轮 analysis-5 的默认目标。

### 2.3 与刺激筛选直接相关的最小约束

为避免前面的刺激筛选与后面的正式实验脱节，至少保留以下约束：

- 正式刺激面板暂定为 30 条正式 trial；
- 可另行加入少量练习 trial，但练习部分不进入 analysis-5 的主筛选结果；
- 正式 trial 的呈现顺序应在被试内随机化；
- 正式 trial 默认不提供逐题反馈，以免改变后续 trial 的独立性；
- 所有正式 trial 应使用同一套目标节点 belief 报告界面，不因结构类型不同而切换反应形式。

这些约束的作用，是让当前筛出的 top-30 panel 可以直接作为后续正式实验的刺激来源，而不是只在 analysis 内部成立、落到真实任务时又需要重新解释。

### 2.4 本轮不在这里展开的内容

以下内容可以在后续实验实现阶段再细化，但目前不作为 analysis-5 的核心决策项：

- 具体界面布局与视觉风格；
- 练习 trial 的数量与讲解方式；
- belief ring 的具体视觉编码与交互细节；
- block 划分、休息安排与被试招募规模；
- 反应时清洗与排除规则。

本部分当前只要求做到一件事：明确 analysis-5 筛出的刺激面板，是为“目标节点连续 belief 报告”这一最小任务接口服务的。

## 3. 被试数据格式与拟合

### 3.1 本部分的定位

与第 2 部分相同，这里不提前展开完整的人类数据分析方案，而只明确后续真实被试数据至少应以什么格式保存，以及 analysis-5 当前默认与哪一种拟合接口保持一致。

### 3.2 默认反应格式

当前默认的被试反应格式为 continuous belief report，即每个 trial 记录被试对目标节点三种颜色相对权重的连续报告，而不是只记录一个最终选择的颜色类别。

这样设定有三个直接好处：

- 它与第 2 部分中的 belief ring 任务接口一致；
- 它能更直接保留被试对不确定性的表达，而不是把信息提前压缩为单一颜色；
- 它使 synthetic recovery 可以直接围绕目标节点的完整 belief 分布来生成与识别。

若后续需要增加离散选择版本，可以作为附加分析保留，但不应替代本轮 analysis-5 的主拟合接口。

### 3.3 最小数据记录要求

未来真实被试数据至少应能按 trial 级别记录以下信息：

- `participant_id`
- `trial_index`
- `condition_id`
- `graph_id`
- `canonical_colour_template_id` 或等价标识
- `strength_assignment_id` 或等价标识
- `response_belief_r`
- `response_belief_g`
- `response_belief_b`
- `response_pointer_x`
- `response_pointer_y`
- `response_time`

如条件允许，还应保留能够回连到结构描述的字段，例如 rooted-shell 结构类别、occupancy profile、loop support class 与 tail 标记。这样做的目的，是使后续拟合与误差分析能够直接追溯到某类结构或某组条件，而不是只剩下匿名 trial 编号。

### 3.4 默认拟合方式

analysis-5 当前默认采用 family-level belief-report likelihood 拟合。

具体而言，对每一位被试：

1. 对每个 trial 读取候选 family 在目标节点上的预测分布；
2. 将被试在 belief ring 上报告的三色权重转换为一个标准化 belief 向量；
3. 在一个与该报告格式兼容的观测模型下，计算该 belief 向量在各 candidate family 预测分布下的似然；
4. 对全部 trial 的对数似然求和；
5. 以总 log-likelihood 最高的 family 作为该被试的最佳拟合结果。

当前更自然的做法，是使用以 candidate family 预测分布为中心的 continuous-report likelihood；若实现上需要一个具体分布族，可优先考虑 Dirichlet 型观测模型或与之等价的 simplex 噪声模型。若后续额外比较离散 choice 拟合版本，可以作为补充分析输出；但当前 analysis-5 的主 recovery 结果与主刺激筛选逻辑，均应与这一 belief-report fitting 接口保持一致。

### 3.5 对第 1、4 部分的约束意义

之所以在本轮 TODO 中保留这一最小拟合接口，是因为它会反过来约束前面的刺激筛选与后面的 recovery：

- 第 1 部分中的条件评分，应服务于 family-level belief discrimination；
- 第 4 部分中的 synthetic recovery，应优先模拟与 continuous belief report 一致的观测过程；
- 若未来正式实验改用别的 response format，则应在后续版本中明确说明：哪些筛选标准需要重算，哪些 recovery 结果需要重做。

换言之，本轮 analysis-5 并不需要把人类实验和拟合写成完整操作手册，但必须把接口说清楚，以免刺激筛选、recovery 和真实数据分析彼此脱节。

## 4. 合成模型恢复

### 4.1 本部分要回答的问题

刺激筛选不能停留在“静态评分较高”这一层面。对于 analysis-5 来说，更重要的问题是：

> 若用筛出的条件面板生成 synthetic data，这套面板是否真的更容易把真实生成的候选 agent family 恢复出来？

因此，合成模型恢复在本分析中的地位不是可选的附加验证，而是检验刺激面板是否具有实际识别价值的核心步骤。一个条件即使在静态评分上表现良好，如果在 recovery 中仍然系统性混淆关键 family 边界，就不能视为真正成功的候选刺激。

### 4.2 recovery 的基本对象

recovery 的对象是已经经过结构筛选、颜色模板合法化、强度分配与条件评分之后得到的候选条件面板，而不是单个孤立条件。

至少应对以下类型的面板进行 recovery 比较：

1. 一个简化 baseline panel，用于表示不引入当前 analysis-5 复杂设计时的基本识别水平；
2. analysis-5 的正式候选面板，即基于 rooted-shell 结构、合法 canonical colour template 与异质 strength assignment 组装出的 top panel。

若计算资源允许，还应增加以下对照：

3. 只按分数直接截取得到的 pure-score panel；
4. 在分数基础上施加结构多样性约束后得到的 diversity-aware panel。

### 4.3 synthetic data 的生成逻辑

recovery 的 synthetic data 应由候选 agent family 在给定面板上生成，而不是由人为指定“正确答案”。

具体而言，应对 Analysis 4 的 6-family 候选 agent 集合分别进行如下过程：

- 固定某一个 family 为生成模型；
- 在给定条件面板上，读取该 family 对目标节点的预测分布；
- 基于这些分布生成 synthetic participant 或 synthetic trial responses；
- 再用 analysis-5 规定的 family 评分与拟合方法，对 synthetic data 进行反向识别；
- 比较“生成 family”与“恢复出的 family”是否一致。

这样得到的 recovery 结果，才真正对应“这套刺激能否支持 family-level diagnosis”这一科学问题。

### 4.4 recovery 至少应报告的结果

每一类候选面板至少应报告以下结果：

- confusion matrix
- diagonal recovery rate
- family-specific recovery summary
- 平均 log-likelihood gap 或其他反映恢复稳定性的 summary

如果计算预算允许，还应进一步报告：

- 不同 panel size（如 10 / 20 / 30）下的 recovery 曲线；
- 困难 family 边界的 pair-specific recovery 结果；
- 哪些具体条件最有助于拉开 `M_exact` vs `M_cluster`、`M_rho` vs `M_rho,beta`、`M_0` vs `M_star` 等关键混淆边界。

### 4.5 recovery 与主评分的关系

recovery 不是主评分的重复，而是对主评分的一次外部检验。

在解释时应明确区分：

- 主评分回答的是：某个条件在静态预测层面是否具有较强的 family separability；
- recovery 回答的是：由这些条件组成的面板，在模拟行为数据层面是否真的更容易恢复出正确 family。

理想情况下，好的面板应同时满足这两点；但若两者出现偏离，应优先以 recovery 结果修正对面板优劣的判断。

### 4.6 噪声模型与拟合兼容性

recovery 中使用的噪声模型与拟合规则，应尽量与 analysis-5 后续拟用于真实被试数据的分析方法兼容，而不应为了得到更漂亮的 recovery 数字而单独设计一套过于理想化的 synthetic observation model。

若使用 continuous belief report 作为主拟合形式，则 recovery 应优先采用与 belief-report likelihood 一致的 synthetic 生成与识别方式。若额外比较离散 choice 版本，也应明确说明它在本分析中的地位只是补充，而不是替代主 recovery 结果。

凡是涉及 concentration、trial 数量、choice noise 或 missingness 机制的设定，都必须在输出文件与最终报告中写清楚，便于后续重复与比较。

## 5. 报告、图表与交付物

### 5.1 报告要回答的问题

analysis-5 的最终报告不应只是条件表、分数表或若干图的堆积，而应明确回答以下问题：

1. rooted-shell 结构语言是否比单纯按节点数组织更适合描述本轮刺激筛选；
2. 合法 canonical colour template 的重新构造，是否修复了旧刺激模板中的关键问题；
3. 哪些结构类、颜色模板与强度组合最有利于区分 Analysis 4 的 6-family 候选 agent；
4. 最终 top panel 为什么被选中，而不是其他高分但结构冗余或恢复效果较差的条件；
5. recovery 结果是否支持将这套面板作为后续人类实验的正式刺激来源。

### 5.2 报告应包含的主要内容

最终 written report 至少应包含以下部分：

- 科学问题与设计动机；
- 基础图集如何生成，以及为什么采用“结构覆盖优先、少节点优先”的策略；
- 合法颜色模板如何生成、压缩并选出 canonical template；
- strength assignment 如何展开，以及候选条件如何去重与评分；
- top panel 的组装原则与结构覆盖情况；
- synthetic model recovery 的结果与解释；
- 结论：当前面板的优点、残留混淆边界，以及下一步最需要补强的地方。

### 5.3 图表要求

最终报告至少应包含以下几类图表：

1. **结构概览图**：说明 rooted-shell 结构语言与主要结构类别；
2. **基础图集分布图或汇总表**：展示节点数、环节点数分布、环间连接模式、中继冗余度、环支撑类型、tail 等覆盖情况；
3. **颜色模板与强度示意图**：展示合法 canonical colour template 与 strength assignment 的含义；
4. **最终 top panel 汇总表**：说明每条条件属于何种结构类、采用何种颜色模板和强度模式，以及其主要评分指标；
5. **recovery 结果图表**：至少包括 confusion matrix 与 family-specific recovery summary；
6. **关键边界解释图**：用于解释 `M_exact` vs `M_cluster`、`M_rho` vs `M_rho,beta`、`M_0` vs `M_star` 等主要混淆边界。

这些图表的作用不仅是展示结果，还应承担“让后续执行者看懂这套刺激为什么被保留”的解释任务。

### 5.4 本分析应导出的文件

本分析至少应导出以下 machine-readable 或可直接阅读的文件：

- 基础图库概览（base graph library summary）
- 根壳结构模板目录（rooted-shell structure template catalog）
- 合法颜色模板目录（legal colour-template catalog）
- 标准颜色模板选择记录（canonical colour-template selection summary）
- 证据强度候选方案汇总（strength-assignment candidate summary）
- 最终条件评分排序表（final condition ranking table）
- 前 30 项条件面板文件（top-30 panel file）
- recovery summary
- recovery detail tables
- 一份明确解释“为什么这 30 条被选中”的 written report

若后续继续沿用 analysis 系列的工作链，还应补齐与报告相对应的 README 或等价说明文件，使后来者能够从输出文件直接理解每个阶段的含义。

### 5.5 成功标准

本轮 analysis-5 可视为成功，当且仅当它至少满足以下条件：

1. 构造出一套以 rooted-shell 结构差异为主轴、而不是以总节点数为主轴的基础图集；
2. 重新生成并固定了合法的 canonical colour template，避免相邻节点 argmax 颜色冲突；
3. 在 canonical colour template 之上，系统地生成并筛选了异质 strength assignment；
4. 得到了一个可 machine-read 的 top-30 条件面板；
5. 该面板在 6-family 主评分上具有明确的区分价值；
6. 该面板在 synthetic model recovery 上优于相应 baseline，或至少能明确指出哪些 family 边界仍然是主要瓶颈；
7. 最终报告能够清楚解释“为什么是这 30 条”，而不只是给出结果本身。

### 5.6 实现时仍需坚持的底线

在写代码、出图和写报告时，仍应坚持以下底线：

- 不要重新退回到按节点数分章节或分图组织结果；
- 必须生成新的 rooted-shell 三环基础图集，而不是复用旧图集后仅重新标注 condition；
- 不要直接继承旧分析中的 `preferred_color_template` 或其他旧颜色模板；
- 不要把随机采样当作默认主生成方式；
- 不要只给出 pure-score top panel 而不报告结构覆盖与 recovery 表现；
- 不要让补充性的 pair-specific 指标取代 6-family 主评分。
